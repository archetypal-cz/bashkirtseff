### Key Design Decisions

1. **3 translators in parallel** — one per carnet, maximizing throughput
2. **OPS reviewers with 3 passes** — naturalness (no French), semantic accuracy (with French), deep linguistic check (Opus subagent)
3. **Russian influence detection** — explicit checklist of common russianisms provided to all agents
4. **One agent per carnet lifecycle** — fresh context prevents compaction failures
5. **No RSR/LAN agents** — source preparation already complete for all carnets

### Conductor Tool Limitation

All three conductors reported they could not write `conductor_approved: true` to frontmatter or add CON comments because the `conductor` subagent type only has Read/Grep/Glob tools (no Edit/Write). The ED applied frontmatter updates manually via sed. **Proposed fix**: Either give conductor subagent type Edit access, or document this as expected and have ED handle it.

## Issues Encountered

### Text Corruption (WATCHLIST: encoding issues)
- ops-007 found "баrilася" in 1873-07-14 — a mixed Cyrillic/Latin encoding corruption. Fixed to proper Ukrainian.

### False Friends (WATCHLIST: semantic shifts)
- "air" (expression/manner) translated as "повітря" (atmospheric air) in 008. Classic French→Ukrainian false friend caught by OPS.
- "Змалку" (since childhood) used where "З ранку" (in the morning) was intended in 006.

### Conductor Edit Access
- Conductors spawned as `conductor` subagent type lack Edit/Write tools. This is by design (read-only review) but means the ED must apply frontmatter changes. Not a bug, but worth documenting.

## Observations

### Quality Trends
- Ukrainian quality (0.93-0.95) is on par with the best Czech runs (0.93-0.95 for carnets 005-008)
- Voice preservation scores consistently highest (0.94-0.96) — Marie's personality translates powerfully into Ukrainian

### What Worked Well
1. **Russian influence checklist** — providing explicit examples of russianisms to avoid was highly effective (zero found)
2. **3-pass OPS review** — the deep linguistic subagent pass caught issues the first two passes missed
3. **Parallel execution** — 3 translators + 3 reviewers + 3 conductors maximized throughput
4. **One agent per carnet** — no context exhaustion, no stuck agents, clean lifecycle management
5. **Self-correcting translation** — translators spawning naturalness critics before submitting

### What Could Improve
1. **TranslationMemory could be richer** — the UK TM has good foundations but would benefit from more entries from carnets 006-008
2. **Conductor tool access** — either give Edit or accept ED handles frontmatter
3. **Dialogue formatting** — some entries use `---` (3 hyphens) vs `—` (em dash) for dialogue. Should standardize.
