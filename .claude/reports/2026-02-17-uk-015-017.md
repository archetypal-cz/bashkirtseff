### OPS-016 Stall Investigation

Two consecutive `opus-editor` subagent type agents stalled on carnet 016 with zero progress. Both showed idle notifications but never read or edited any files. A `general-purpose` subagent type successfully completed the same task.

**Hypothesis**: The `opus-editor` subagent type may have a systemic issue in this particular team context — possibly related to context loading, skill file parsing, or team mailbox integration. The three `opus-editor` agents for carnets 015 and 017 worked fine, so this may be timing/resource-related rather than a fundamental problem.

**Workaround**: When opus-editor agents stall, spawn a general-purpose agent with the same instructions.
